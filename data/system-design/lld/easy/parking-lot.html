Parking Lot System Design

Design a parking lot system with classes, enums, strategy pattern, and clean OOP architecture.

#LLD #OOP #design-patterns #strategy-pattern #Java

Problem Statement

Design a Parking Lot System that supports:

  • Multiple floors with different spot types (Compact, Large, Handicapped)
  • Vehicle types: Car, Truck, Motorcycle
  • Automated ticket generation on entry
  • Hourly rate-based payment calculation
  • Real-time spot availability tracking
  • Multiple entry/exit points

Key Classes

📦 ParkingLot
- floors: List<ParkingFloor> - entrances: List<Gate> + parkVehicle(Vehicle): Ticket + unparkVehicle(Ticket): Payment
🅿️ ParkingSpot
- spotId: String - type: SpotType - isAvailable: boolean + assignVehicle(Vehicle): void
🚗 Vehicle
- licensePlate: String - type: VehicleType # Abstract class + Car, Truck, Motorcycle
🎫 Ticket
- ticketId: String - entryTime: LocalDateTime - spot: ParkingSpot + calculateDuration(): long

Design Decisions

01

Singleton Pattern for ParkingLot

Only one instance of the parking lot should exist. Thread-safe lazy initialization ensures safety in concurrent environments.

02

Strategy Pattern for Pricing

Different vehicle types can have different pricing strategies, making it easy to add new pricing models without modifying existing code.

03

Enum for Vehicle & Spot Types

Using enums ensures type safety and prevents invalid states. Each vehicle type maps to compatible spot types.
